redis简单动态字符串

redis

redis简单动态字符串

redis的SDS分为连部分，头+数据字段

SDS有多种头部，不同的长度分配的头部不同。

头部包含

  • len：以使用的长度
  • alloc：头部+后面分配的+\0的结束符长度（实际分配的大小）
  • flags：头部的类型
  • buf[]：柔性数组，标识后面是一个char指针，不占大小

初始化

初始化时，传入字符串，会计算字符串的长度，然后进行初始化

初始化：

  1. 计算类型
  2. 计算头长度
  3. 分配内存：头+字符串长度+1 (\0)
  4. 全部初始化为0
  5. 初始化头部数据
  6. 把字符串拷贝到SDS的字符串位置

字符串修改

结尾增加字符串：

  1. 计算当前字符串长度
  2. 确保有足够的空余空间，不够会扩充
    1. 传入字符串和新增长度
    2. 如果新旧字符串长小于1M，长度扩大2倍，否则加1M
    3. 计算字符串类型（为了压缩空间）
    4. 如果头不变，从原开始重新分配空间，数据不会修改，只是增大空间；如果头变，分配新的空间，将旧字符串拷贝
  3. 新字符串追加
  4. 设置新的长度

字符串修剪：

  1. 从左到右和从右到左找满足要修剪的子串
  2. 计算边界和剩余长度
  3. 剩余字符串移动到头部
